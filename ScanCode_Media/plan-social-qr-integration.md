# 计划：GitHub Pages 社交媒体二维码集成方案

> **目标**: 在 https://matthewhemhgz-dev.github.io/ 上优雅地展示各平台关注入口，解决「只有二维码如何让用户一键添加」的问题。

---

## 一、调研结论

### 1.1 各平台关注能力对比

| 平台 | 直接 URL | 外部可跳转 | 二维码 | 推荐方案 |
|------|----------|:-----------|:-------|---------|
| **微信公众号** | ❌ `mp/profile_extaction` 仅微信内有效 | ❌ | ✅ 唯一可靠方式 | **二维码 + 引导文案** |
| **微信视频号** | ⚠️ `weixin://finder/{id}` 需装微信 | ⚠️ URL Scheme 不稳定 | ✅ 最可靠 | **二维码（与公众号共用或独立）** |
| **小红书** | ✅ `xhslink.com/a/{code}` 或 `/user/profile/{id}` | ✅ 可直接跳转 | ✅ | **直接链接 + 备用二维码** |
| **抖音** | ✅ `v.douyin.com/{code}` 或 `/user/{uid}` | ✅ 可直接跳转 | ✅ | **直接链接 + 备用二维码** |

### 1.2 关键发现

**微信公众号/视频号 — 只有二维码是可靠的：**
- 公众号的 `mp.weixin.qq.com/mp/profile_extaction?__biz=xxx` 链接**只在微信内有效**，浏览器打开会报错
- 视频号的 `weixin://finder/` 是 URL Scheme，依赖设备是否安装微信
- **结论：对于外部网站（GitHub Pages），二维码是唯一 100% 可用的方案**

**小红书/抖音 — 有直接链接可用：**
- 小红书分享链接格式：`https://www.xiaohongshu.com/user/profile/{userId}` 或短链 `https://xhslink.com/a/{随机码}`
- 抖音分享链接格式：`https://v.douyin.com/{短码}` → 浏览器打开后自动跳转到个人主页
- **结论：可以直接用 `<a>` 标签跳转，用户在手机上点击会唤起对应 App**

### 1.3 业界最佳实践

来源: QR Code Design Best Practices (2025), CSDN, 新榜等

1. **点击弹出大图 Modal** — 最主流方案（CTR 提升 15-40%）
2. **CTA 文案不可少** — 「扫码关注」「扫码添加」比裸二维码效果好
3. **尺寸规范**:
   - 页面内缩略图: 80-120px
   - 弹出大图: 256-320px（方便手机扫描）
   - 安全区（Quiet Zone）: 二维码周围留白 ≥ 边长的 1/3
4. **配色**: 深色二维码 + 浅背景（保证高对比度），可融入品牌色
5. **位置策略**:
   - Hero 区域 CTA 旁边（首屏可见）
   - Footer 社交区（全局可达）
   - 独立 Contact/Follow 页面（完整展示）

## 二、推荐方案：混合模式

### 核心思路

```
┌─────────────────────────────────────────────┐
│  GitHub Pages 网站                            │
│                                             │
│  ┌─────────┐  ┌─────────┐  ┌─────────┐     │
│  │ 📱公众号  │  │ 📕小红书 │  │ 🎵抖音   │     │
│  │ [点击→弹窗]│  │ [直接跳转]│  │ [直接跳转]│     │
│  │  大二维码  │  │          │  │          │     │
│  └─────────┘  └─────────┘  └─────────┘     │
│                                             │
│  ┌─────────┐                                 │
│  │ 📺视频号  │  （可选：与公众号合并展示）      │
│  │ [点击→弹窗]│                                │
│  └─────────┘                                 │
└─────────────────────────────────────────────┘
```

### 方案细节

#### A. 微信公众号 / 视频号 → 点击弹出二维码 Modal

**交互流程**：
1. 用户点击「公众号」图标/按钮
2. 弹出 Modal（居中遮罩层）
3. 显示大尺寸二维码（300x300px）
4. 显示引导文案：「微信扫一扫关注」+ 账号名「祈研所」
5. 点击遮罩层或关闭按钮关闭 Modal

**为什么不用直接链接**：
- 微信外部的公众号链接无效
- URL Scheme (`weixin://`) 在桌面浏览器无反应
- 二维码是唯一跨平台、100% 可靠的方案

#### B. 小红书 / 抖音 → 直接链接跳转

**实现方式**：
```html
<!-- 小红书 -->
<a href="https://www.xiaohongshu.com/user/profile/{userId}" target="_blank" rel="noopener">
  <span class="icon">📕</span> 小红书
</a>

<!-- 抖音 -->
<a href="https://v.douyin.com/{shortCode}" target="_blank" rel="noopener">
  <span class="icon">🎵</span> 抖音
</a>
```

**用户体验**：
- 手机浏览器点击 → 自动唤起 App 并跳转到个人主页
- 桌面浏览器点击 → 打开网页版个人主页
- **备用**：如果用户没有安装 App 或链接失效，提供二维码作为 fallback

#### C. 统一视觉风格

所有社交入口使用一致的设计语言：
- 圆形图标 + 平台名称
- Hover 效果：品牌色边框发光（祖母绿/琥珀金）
- 品牌一致性：使用 Amber Geek 配色系统

### Step 2: 创建 Astro 组件

新建文件: `src/components/SocialQR.astro`

组件功能：
- Props: `platform`, `qrImage`, `directUrl`, `name`
- 渲染逻辑：
  - 有 `directUrl` → 渲染为 `<a>` 链接
  - 无 `directUrl`（仅有二维码）→ 渲染为按钮，点击触发 Modal
- Modal 内部：大图二维码 + CTA 文案 + 关闭按钮
- 响应式：移动端全屏 Modal，桌面端居中浮层

### Step 3: 集成到网站各位置


#### 3.2 Footer 社交区域

在页脚添加完整的社交矩阵：

```

📱 公众号(扫码) | 📕 小红书 | 🎵 抖音 | 🐙 GitHub | 📧 Email
```

#### 3.3 Contact 页面（如有）

创建或更新联系页面，集中展示所有联系方式和二维码。

### Step 4: 样式与品牌对齐

CSS 要点：
- Modal 遮罩层: `rgba(42, 42, 42, 0.8)` (charcoal)
- Modal 内容区: `#F7F3EE` 背景 (oatmeal grey)
- 关闭按钮: 祖母绿 `#2E7D5C`
- 二维码边框: 8px 圆角 + 琥珀金 `#E5A93C` 细边框
- 动画: fade-in 200ms ease-out
- 移动端适配: Modal 宽度 90vw，二维码自适应

## 四、备选方案对比

| 方案 | 优点 | 缺点 | 推荐度 |
|------|------|------|--------|
| **A. 混合模式（推荐）** | 微信用二维码，其他用链接；体验最优 | 需要准备素材 | ⭐⭐⭐⭐⭐ |
| B. 全二维码 | 统一体验，无需获取链接 | 用户需逐个扫码，操作成本高 | ⭐⭐⭐ |
| C. 全链接 | 最简洁 | 微信公众号/视频号链接在外部无效 | ⭐⭐ |
| D. Linktree 中转页 | 一个链接聚合所有平台 | 多一次跳转，增加流失 | ⭐⭐⭐ |
| E. 小程序/中间页 | 可以做数据统计 | 开发成本高，维护复杂 | ⭐⭐ |
