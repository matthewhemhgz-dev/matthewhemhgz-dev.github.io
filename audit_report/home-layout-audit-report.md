## 二、发现的布局问题

### 问题 1: 移动端导航栏冲突

**严重程度：** 中等  
**影响范围：** 移动端 (<=768px)  
**问题描述：**  
在 `Navigation.astro` 中，存在移动端菜单的 CSS 样式冲突。`.nav-wrapper.nav-collapsed` 在移动端的处理逻辑与桌面端不一致，可能导致菜单显示异常。

**根因分析：**  
在 `navigation.css` 第 267-277 行，对移动端的 `nav-collapsed` 状态没有正确禁用桌面端的行为，可能导致菜单在移动端无法正常展开。

**建议修复：**  
```css
@media (max-width: 768px) {
  .nav-wrapper.nav-collapsed {
    padding: 0 var(--qi-container-padding, 24px);
  }
  
  .nav-wrapper.nav-collapsed:hover .nav-links {
    opacity: 0;
    width: 0;
    overflow: hidden;
  }
}
```

---

### 问题 2: 英雄区域浮动卡片定位

**严重程度：** 中等  
**影响范围：** 中等屏幕 (768px-1200px)  
**问题描述：**  
在 `home-hero.css` 第 364-374 行，`.hero-extra` 使用绝对定位，在中等屏幕尺寸下可能导致内容溢出或与主内容重叠。

**根因分析：**  
`.hero-extra` 的定位计算基于固定的 100px 偏移量，在不同屏幕尺寸下无法自适应调整。

**建议修复：**  
```css
.hero-extra {
  position: absolute;
  top: 15%;
  left: clamp(50% + 50px, 55%, 50% + 150px);
  width: clamp(280px, 30%, 340px);
  /* 其他样式保持不变 */
}
```

---

### 问题 3: 响应式断点重复

**严重程度：** 轻微  
**影响范围：** 代码维护性  
**问题描述：**  
在 `home-hero.css` 和 `home-responsive.css` 中存在重复的响应式断点定义，可能导致样式覆盖问题。

**根因分析：**  
1. `home-hero.css` 第 399-414 行定义了 `@media (max-width: 1200px)`
2. `home-responsive.css` 中也有类似的断点定义
3. 4K 相关的断点在两个文件中都有重复

**建议修复：**  
统一管理断点，将响应式样式集中在 `home-responsive.css` 中。

---

### 问题 4: 首页布局网格缺少清晰的响应式处理

**严重程度：** 中等  
**影响范围：** 平板和中等屏幕  
**问题描述：**  
首页的 `.home-layout-grid` 布局在 `index.astro` 第 153-208 行定义，但缺少明确的响应式回退处理，在中等屏幕尺寸下可能出现布局错乱。

**根因分析：**  
缺少对 `.home-layout-grid` 在不同断点下的明确 grid-template-columns 定义。

**建议修复：**  
```css
@media (max-width: 1024px) {
  .home-layout-grid {
    grid-template-columns: 1fr;
  }
  
  .home-layout-sidebar {
    order: 2;
  }
}
```
