# 祈研所网站优化 - 产品需求文档

## 概述

- **摘要**：基于多专家第二轮分析报告，对祈研所网站进行迭代优化，重点包括 hero-float-card 重构、主题模式对比度优化、品牌设计风格统一以及其他问题的改进。
- **目的**：解决分析报告中识别的问题，提升网站的整体质量和用户体验，确保符合品牌设计风格。
- **目标用户**：全球范围内的技术爱好者、知识管理从业者、项目管理者和IT/AI领域专业人士。

## 目标

- 重构 hero-float-card 组件，使其与整体设计风格一致
- 优化底部黑白两种模式的逻辑显示，确保正确的对比度
- 统一品牌设计风格，确保所有元素符合品牌视觉标准
- 解决分析报告中识别的其他问题
- 提升网站的整体用户体验和专业度

## 非目标（范围外）

- 完全重构网站架构
- 更改网站的核心功能和内容结构
- 引入新的第三方依赖
- 大幅修改现有的内容布局

## 背景与上下文

- 网站当前使用 Astro 6 + Tailwind CSS + 设计令牌系统
- 已部署在 GitHub Pages
- 支持多语言（中文/英文）和亮色/深色模式
- 多专家第二轮分析报告已识别出多个需要改进的问题
- 品牌设计风格需要明确和统一

## 功能需求

- **FR-1**：重构 hero-float-card 组件，使其与整体设计风格一致
- **FR-2**：优化底部黑白两种模式的逻辑显示，确保正确的对比度
- **FR-3**：统一品牌设计风格，确保所有元素符合品牌视觉标准
- **FR-4**：添加社交媒体二维码扫描添加功能
- **FR-5**：优化微信分享二维码在移动端的显示时间
- **FR-6**：统一社交媒体图标风格
- **FR-7**：为英文页面提供英文社交媒体平台链接

## 非功能需求

- **NFR-1**：性能优化，确保页面加载速度快，构建产物大小在预算范围内
- **NFR-2**：视觉设计提升，确保所有元素符合品牌设计风格
- **NFR-3**：响应式设计，确保在所有设备上的显示效果良好
- **NFR-4**：无障碍性增强，确保所有用户都能正常使用网站
- **NFR-5**：代码质量优化，提高可维护性和可读性

## 约束

- **技术**：基于现有的 Astro 6 + Tailwind CSS 技术栈
- **业务**：保持网站的现有主题和内容结构
- **依赖**：使用现有的第三方库和工具

## 假设

- 品牌设计风格为用户的个人品牌风格，而非 Amber Geek 设计风格
- 所有社交媒体平台账号已存在且可访问
- 网站的基本架构和功能保持不变

## 验收标准

### AC-1：hero-float-card 重构

- **Given**：用户访问网站的首页
- **When**：用户查看 hero 区域
- **Then**：hero-float-card 组件显示正常，与整体设计风格一致
- **验证**：`human-judgment`

### AC-2：底部黑白模式对比度

- **Given**：用户切换网站主题
- **When**：用户在亮色和暗色模式之间切换
- **Then**：底部区域的文字和背景对比度正确，在两种模式下都清晰可见
- **验证**：`human-judgment`

### AC-3：品牌设计风格统一

- **Given**：用户浏览网站的不同页面
- **When**：用户观察网站的视觉元素
- **Then**：所有元素符合品牌设计风格，视觉一致性良好
- **验证**：`human-judgment`

### AC-4：社交媒体二维码扫描添加功能

- **Given**：用户访问网站的页脚
- **When**：用户查看社交媒体链接
- **Then**：用户可以通过扫描二维码添加社交媒体账号
- **验证**：`programmatic`

### AC-5：微信分享二维码优化

- **Given**：用户在移动端访问博客文章
- **When**：用户点击微信分享按钮
- **Then**：二维码显示时间足够用户扫描（至少5秒）
- **验证**：`programmatic`

### AC-6：社交媒体图标风格统一

- **Given**：用户访问网站的页脚
- **When**：用户查看社交媒体图标
- **Then**：所有图标风格一致，符合品牌设计标准
- **验证**：`human-judgment`

### AC-7：英文页面社交媒体链接

- **Given**：用户访问英文版本的网站
- **When**：用户点击社交媒体链接
- **Then**：链接指向英文社交媒体平台
- **验证**：`programmatic`
