# 祈研所网站架构与设计迭代路线分析框架

## 研究概述
- **研究主题**: 祈研所网站架构迭代路线分析 — 博客与项目分离评估及页面层次系统设计
- **范围**: 祈研所网站 (QiLab)，当前版本，中文/英文双语
- **分析领域**: 网站架构设计、用户体验、信息架构
- **核心研究问题**:
  1. 博客文章与项目展示是否应该分离？当前整合方式是否造成用户混淆？
  2. 背景光效浮动粒子效果丢失的技术原因与恢复方案？
  3. 如何系统性设计页面层次结构，提升整体视觉体验？
  4. 非首页模块（博客详情、项目页、关于页等）的设计优化方向？

## 框架选择

| 章节 | 选择的框架 | 应用方式 |
|------|------------|----------|
| 信息架构评估 | 用户旅程地图 (User Journey Map) + 信息层级分析 | 分析用户浏览路径，识别混淆点 |
| 功能模块分离决策 | SWOT分析 + 卡诺模型 | 评估博客与项目分离的优缺点和用户需求优先级 |
| 视觉层次设计 | 视觉层次理论 (Visual Hierarchy) + 格式塔原理 | 系统性设计页面层次结构 |
| 技术实现分析 | 技术栈评估框架 | 分析粒子效果丢失原因及恢复方案 |
