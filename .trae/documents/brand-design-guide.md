# 品牌设计指南

## 1. 品牌概述

### 1.1 品牌定位
- **品牌名称**：祈研所 (Qi Lab)
- **品牌定位**：专注于知识管理、人工智能和创新技术的研究机构
- **品牌价值观**：创新、专业、协作、开放

### 1.2 品牌使命
- 探索前沿技术，推动知识管理创新
- 构建智能、高效的知识管理系统
- 为用户提供专业、实用的知识管理工具和解决方案

## 3. 动效系统

### 3.1 动画原则
- **流畅性**：动画应该流畅自然，避免突兀的变化
- **目的性**：动画应该有明确的目的，增强用户体验
- **一致性**：动画风格应该在整个网站中保持一致
- **性能**：动画应该高效，不影响页面性能

### 3.2 动画类型

#### 页面过渡
- **淡入淡出**：用于页面加载和切换
- **滑动**：用于内容切换和导航
- **缩放**：用于模态框和弹出层

#### 交互反馈
- **悬停效果**：用于按钮、链接和卡片
- **点击效果**：用于按钮和可点击元素
- **加载动画**：用于数据加载和处理

#### 背景动效
- **粒子系统**：用于首页和重要页面的背景
- **流体效果**：用于动态背景
- **生成艺术**：用于创意展示

### 3.3 动画参数
- **持续时间**：150ms (快速) - 500ms (慢速)
- **缓动函数**：cubic-bezier(0.25, 0.1, 0.25, 1)
- **延迟**：0ms - 200ms (用于序列动画)

## 4. 组件设计

### 4.1 按钮

#### 主要按钮
- **背景色**：翡翠绿 (#4ade80)
- **文字色**：白色 (#ffffff)
- **悬停效果**：背景色变深，添加轻微阴影
- **点击效果**：轻微缩放，背景色进一步变深

#### 次要按钮
- **背景色**：透明
- **边框色**：翡翠绿 (#4ade80)
- **文字色**：翡翠绿 (#4ade80)
- **悬停效果**：背景色变为浅翡翠绿，文字色变深

#### 禁用按钮
- **背景色**：浅灰色 (#e2e8f0)
- **文字色**：中灰色 (#94a3b8)
- **鼠标指针**：not-allowed

### 4.2 卡片

#### 标准卡片
- **背景色**：白色 (#ffffff)
- **边框**：1px 浅灰色 (#e2e8f0)
- **圆角**：0.5rem (8px)
- **阴影**：小阴影
- **悬停效果**：添加中等阴影，轻微上浮

#### 特色卡片
- **背景色**：白色 (#ffffff)
- **边框**：1px 薄荷绿 (#a7f3d0)
- **圆角**：0.75rem (12px)
- **阴影**：中等阴影
- **悬停效果**：添加大阴影，轻微上浮，边框色变为翡翠绿

### 4.3 表单元素

#### 输入框
- **背景色**：白色 (#ffffff)
- **边框**：1px 浅灰色 (#e2e8f0)
- **文字色**：深灰色 (#334155)
- **聚焦效果**：边框色变为翡翠绿，添加轻微阴影
- **错误状态**：边框色变为红色，显示错误信息

#### 下拉菜单

- **文字色**：深灰色 (#334155)

- **选中效果**：背景色变为薄荷绿 (#a7f3d0)

### 4.4 导航栏

#### 桌面导航

- **文字色**：深灰色 (#334155)
- **活动链接**：翡翠绿 (#4ade80)
- **悬停效果**：文字色变为翡翠绿

#### 移动导航

- **菜单按钮**：三条横线图标

## 6. 响应式设计

### 6.1 断点
- **移动端**：< 640px
- **平板**：640px - 1024px
- **桌面**：1024px - 1280px
- **大屏**：> 1280px

### 6.2 设计原则
- **移动优先**：从移动端开始设计，逐步扩展到更大的屏幕
- **内容优先级**：在小屏幕上优先显示重要内容
- **弹性布局**：使用 Flexbox 和 Grid 确保布局在不同屏幕尺寸下都能正常显示
- **图片响应式**：确保图片在不同屏幕尺寸下都能正确显示
- **字体大小**：在小屏幕上使用较小的字体大小

### 6.3 具体适配
- **移动端**：单列布局，隐藏侧边栏，使用汉堡菜单
- **平板**：双列布局，简化侧边栏，调整间距
- **桌面**：多列布局，完整侧边栏，标准间距
- **大屏**：多列布局，完整侧边栏，增加内容宽度

## 7. 国际化设计

### 7.1 语言支持
- **中文**：简体中文
- **英文**：美国英语

### 7.2 设计原则
- **文本长度**：考虑不同语言文本长度的差异，使用弹性布局
- **文本方向**：支持从左到右的文本方向
- **字体选择**：确保字体支持所有需要的字符
- **日期和时间**：使用本地化的日期和时间格式
- **货币和单位**：使用本地化的货币和单位格式

### 7.3 具体适配
- **导航**：确保导航链接在不同语言下都能正常显示
- **按钮**：确保按钮文本在不同语言下都能适应按钮大小
- **卡片**：确保卡片内容在不同语言下都能正确显示
- **表单**：确保表单标签和提示文本在不同语言下都能正常显示

## 8. 可访问性

### 8.1 设计原则
- **颜色对比度**：确保文本和背景之间有足够的对比度
- **键盘导航**：确保所有功能都可以通过键盘访问
- **屏幕阅读器**：确保所有内容都可以被屏幕阅读器识别
- **语义化 HTML**：使用正确的 HTML 元素和 ARIA 属性
- **焦点管理**：确保焦点状态清晰可见

### 8.2 具体要求
- **颜色对比度**：文本与背景的对比度至少为 4.5:1
- **键盘导航**：所有可交互元素都可以通过 Tab 键访问
- **屏幕阅读器**：所有图像都有 alt 属性，所有表单元素都有标签
- **语义化 HTML**：使用 header、nav、main、section、article、footer 等语义化元素
- **焦点管理**：焦点状态有明显的视觉指示

## 9. 实施指南

### 9.1 设计令牌
- **使用方式**：在 CSS 中使用 CSS 变量引用设计令牌
- **命名规范**：使用 `--qi-` 前缀，遵循 BEM 命名规范
- **更新流程**：修改设计令牌文件，然后更新所有使用该令牌的文件

### 9.2 组件库
- **组件创建**：创建符合设计规范的组件
- **组件复用**：在整个网站中复用组件，确保一致性
- **组件更新**：当设计规范变更时，更新所有相关组件

### 9.3 代码规范
- **CSS 规范**：使用 BEM 命名规范，遵循 CSS 最佳实践
- **JavaScript 规范**：使用 ES6+ 语法，遵循 JavaScript 最佳实践
- **HTML 规范**：使用语义化 HTML，遵循 HTML 最佳实践

### 9.4 测试和验证
- **视觉测试**：确保所有页面在不同屏幕尺寸下都能正确显示
- **功能测试**：确保所有功能都能正常工作
- **可访问性测试**：确保网站符合可访问性标准
- **性能测试**：确保网站加载速度快，动画流畅

### 10.1 设计系统维护
- **定期审查**：定期审查设计系统，确保其符合品牌定位和用户需求
- **持续更新**：根据用户反馈和业务需求，持续更新设计系统
- **文档维护**：保持设计指南文档的更新和完整

### 10.2 品牌一致性
- **品牌审查**：定期审查网站，确保品牌一致性
- **培训**：确保所有参与网站开发的人员了解品牌设计规范

### 10.3 未来发展

- **技术发展**：关注技术发展，适时更新技术实现
