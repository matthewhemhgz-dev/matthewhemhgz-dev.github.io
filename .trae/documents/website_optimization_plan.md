# 网站优化计划

## 1. 仓库研究结论

当前代码库是一个基于 Astro 6 的静态网站，使用 Tailwind CSS 进行样式管理，支持多语言（中文和英文），并部署在 GitHub Pages 上。网站已经实现了大部分功能，但仍有一些优化任务需要完成。

## 2. 需要编辑的文件和模块

### 2.1 已完成的任务

- ✅ 读者评价区域改为历史timeline
- ✅ CTA部分背景改为纯色
- ✅ 两个hero-float-card改为单个
- ✅ 去掉测试的博客文章
- ✅ 修复博客文章的mermaid错误，并禁止在出现

### 2.2 待完成的任务

1. **联系方式如微信，小红书等，滑动到对应区域则出现二维码**
   - 文件：`src/components/global/Footer.astro`

2. **文章的内容边框要回到有呼吸感，沉浸感，与文字有padding**
   - 文件：`src/styles/article.css`

3. **文字排版和样式需要与日夜两种模式的品牌调性一致**
   - 文件：`src/styles/article.css`

4. **修复导航栏日与夜两种模式的样式和逻辑**
   - 文件：`src/components/global/Navigation.astro`
   - 文件：`src/styles/navigation.css`

5. **英语部分的页面内容自动翻译与中文完全同步**
   - 文件：`src/pages/en/index.astro`

## 3. 修改步骤

### 3.1 实现联系方式二维码滚动显示

1. 打开 `src/components/global/Footer.astro`
2. 添加二维码容器和图片
3. 实现滚动检测逻辑，当滚动到联系区域时显示二维码
4. 添加相应的样式

### 3.2 优化文章内容边框和呼吸感

1. 打开 `src/styles/article.css`
2. 更新 `.prose` 类的样式，添加适当的 padding、背景、边框和阴影
3. 确保样式具有呼吸感和沉浸感

### 3.3 实现日夜两种模式的文字排版和样式

1. 在 `src/styles/article.css` 中添加深色模式样式
2. 确保文字排版和样式与品牌调性一致
3. 测试两种模式下的显示效果

### 3.4 修复导航栏日夜模式样式和逻辑

1. 检查 `src/components/global/Navigation.astro` 中的逻辑
2. 更新 `src/styles/navigation.css` 中的深色模式样式
3. 确保导航栏在两种模式下都能正确显示

### 3.5 同步英语页面内容

1. 打开 `src/pages/en/index.astro`
2. 更新网站信息、工具箱分类、平台矩阵等内容，与中文版本保持一致
3. 确保所有部分的内容都已同步

## 4. 潜在依赖和考虑因素

- **响应式设计**：确保所有修改在不同屏幕尺寸下都能正确显示
- **性能优化**：确保修改不会影响网站性能
- **可访问性**：确保修改符合可访问性标准
- **品牌一致性**：确保所有修改都符合品牌调性

## 5. 风险处理

- **样式冲突**：在修改样式时，确保不会与现有样式冲突
- **功能回归**：在修改代码时，确保不会破坏现有功能
- **性能问题**：在添加新功能时，确保不会影响网站性能
- **兼容性问题**：确保修改在不同浏览器中都能正确显示

## 6. 验证步骤

1. 运行 `npm run dev` 启动开发服务器
2. 检查所有修改是否正确实现
3. 测试日夜模式切换是否正常
4. 测试响应式设计是否正确
5. 运行 `npm run build` 确保构建成功
6. 运行 `npm run lint` 确保代码质量
7. 运行 `npm run size-check` 确保构建产物大小在预算范围内

## 7. 预期结果

- 联系方式区域在滚动到对应位置时显示二维码
- 文章内容具有呼吸感和沉浸感，与文字有适当的padding
- 文字排版和样式在日夜两种模式下都与品牌调性一致
- 导航栏在日夜两种模式下都能正确显示
- 英语页面内容与中文页面内容完全同步
